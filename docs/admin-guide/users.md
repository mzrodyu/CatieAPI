# 用户管理

用户管理是 CatieAPI 管理后台的核心模块，第一版必须具备。管理员需要能快速定位用户、处理额度问题、排查调用异常，并在必要时限制账号风险。

## 1. 用户列表

列表字段建议：

- 用户 ID
- 邮箱或用户名
- 角色
- 状态
- 余额或剩余额度
- 今日请求数
- 总请求数
- 注册时间
- 最后登录时间

基础操作：

- 搜索用户
- 按状态筛选
- 按角色筛选
- 查看详情
- 禁用账号
- 解封账号

## 2. 用户详情

详情页建议分组展示：

- 基础信息
- 额度信息
- API Key
- 调用日志
- 登录记录
- 管理员备注

基础信息：

- 用户 ID
- 邮箱或用户名
- 角色
- 状态
- 注册时间
- 最后登录时间
- 注册 IP

## 3. 用户状态

建议状态：

- 正常
- 禁用
- 欠费
- 风控限制

状态影响：

- 正常：允许登录和调用 API。
- 禁用：禁止登录和调用 API。
- 欠费：允许登录，不允许继续调用付费模型。
- 风控限制：允许登录，但限制 API 调用频率或禁止创建新 Key。

## 4. 角色权限

第一版建议角色：

- 用户：管理自己的 API Key、查看用量和账单。
- 管理员：管理用户、渠道、模型、价格和系统配置。

后续可以扩展：

- 运营：管理用户额度、公告和额度流水。
- 只读管理员：只能查看数据，不能修改配置。

## 5. 额度调整

管理员应能执行：

- 增加额度
- 扣减额度
- 设置固定额度
- 查看额度变更记录

每次额度调整必须记录：

- 操作人
- 调整前额度
- 调整后额度
- 调整原因
- 操作时间

## 6. API Key 管理

管理员在用户详情中应能查看：

- Key 名称
- Key 前缀
- 状态
- 创建时间
- 最后使用时间
- 请求次数

管理员应能：

- 禁用用户 Key
- 删除异常 Key
- 查看 Key 相关调用日志

完整 Key 不应在后台明文展示。

## 7. 调用日志

用户详情页需要能按用户过滤调用日志。

建议字段：

- 请求 ID
- API Key 前缀
- 模型
- 渠道
- 状态
- 消耗额度
- 延迟
- 错误码
- 请求时间

## 8. 管理员备注

管理员备注用于记录客服、风控和运营上下文。

要求：

- 支持追加备注。
- 保留备注作者和时间。
- 不允许普通用户查看。
- 不用于系统自动决策。

## 9. 第一版验收标准

- 管理员可以搜索用户。
- 管理员可以查看用户详情。
- 管理员可以调整用户额度。
- 管理员可以禁用和解封用户。
- 管理员可以查看用户 API Key。
- 管理员可以查看用户调用日志。
- 所有关键操作都有审计记录。
