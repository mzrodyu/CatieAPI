# 账号与登录

## 首次初始化

首次打开 CatieAPI 时，点击“控制台”会进入初始化页面。填写：

- 显示名称
- 管理员登录账号
- 邮箱（可选）
- 密码
- Discord 用户 ID（可选）

提交后会创建第一个管理员账号并自动登录。密码使用 bcrypt 哈希保存，不会以明文写入 PostgreSQL 或状态文件。

## 日常登录

初始化完成后，控制台入口会显示登录页面。支持：

- 账号和密码
- 邮箱和密码
- Discord OAuth（管理员启用后）

登录会写入 HttpOnly Cookie。退出后台或账户页后，该 Cookie 会失效。

## 用户注册

管理员可以在“设置 → 账号与注册”中开放或关闭注册。普通用户注册后进入自己的账户页，可以查看余额、模型并创建 API Key，不能访问管理员接口。

## 管理员账号

管理员可以在“设置 → 管理员账号”中修改：

- 登录账号
- 显示名称
- 邮箱
- 密码
- 绑定的 Discord 用户 ID

修改密码必须填写当前密码。Discord 用户 ID 绑定是可选的；绑定后，Discord 登录会恢复对应的本地管理员身份。

## 应急访问

`ADMIN_TOKEN` 是可选的管理 API 应急凭证，不是网页后台的正常登录方式。需要时可在部署环境中设置，并通过请求头使用：

```text
Authorization: Bearer 你的应急管理密钥
```
